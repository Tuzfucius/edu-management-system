## 三、当前架构约束

后端调用链保持：

```text
Controller 接收请求
    ↓
Service 处理业务规则
    ↓
DAO 执行 SQL
    ↓
JdbcTemplate 操作 MySQL
```

包结构保持在：

```text
src/main/java/com/tzufucius/edu/edumanagementsystem
├─ common        通用返回结果、分页对象
├─ entity        单表实体
├─ dto           多表联查展示对象
├─ dao           数据库访问层
├─ service       业务逻辑层
├─ controller    接口控制层
├─ config        Web 配置
├─ interceptor   登录和权限拦截器
└─ exception     全局异常处理
```

继续遵守：

```text
1. 使用 JdbcTemplate，不改 MyBatis、MyBatis-Plus、JPA。
2. 不直接硬删除业务数据，统一通过 status 做逻辑删除。
3. Service 层必须写业务校验，Controller 不直接拼 SQL。
4. 多表展示用 DTO，不把联查字段塞进单表 Entity。
5. 涉及两张以上表同时写入时，Service 方法必须使用 @Transactional。
6. 编号唯一性校验不加 status = 1，保持和数据库唯一约束一致。
```

## 四、下一阶段建议

基础表后端配置已经完成。下一阶段有两个可选方向：

```text
方向 A：先补 Controller 闭环
方向 B：先继续做 sys_user、student、teacher，再统一补 Controller
```

结合当前项目状态，更推荐方向 B：

```text
1. 先完成 sys_user 登录账号基础模块
2. 再完成 student 学生模块
3. 再完成 teacher 教师模块
4. 然后统一实现 Result、GlobalExceptionHandler 和 Controller
```

原因：

```text
1. 学生、教师都需要绑定 sys_user，先做账号模型能减少返工。
2. Controller 暂时不是课程任务的核心风险点，后端业务表和权限链路更关键。
3. 等基础业务稳定后统一补 Controller，可以保持接口返回格式一致。
```
