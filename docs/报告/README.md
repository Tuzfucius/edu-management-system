# 教务管理系统课程设计报告

本目录用于存放教务管理系统课程设计报告正文。报告按照课程目录要求从第 2 章开始撰写，第 1 章“课程设计任务书”不在此处展开。文档采用 Markdown 分级目录组织，章节文件夹与二级标题文件夹共同构成树状结构，便于按章维护、按节检查和后续合并排版。

报告以当前项目实现为依据，技术栈包括 Spring Boot 4.1.0、Java 17、Spring JDBC/JdbcTemplate、MySQL、Vue 3、Element Plus、Axios 与 ECharts。文中只描述架构、接口、数据结构、处理流程、测试方案和运行分析，不粘贴任何后端或前端源代码。涉及程序清单时，只列数据成员、函数名、返回值、参数及调用关系说明。涉及图形说明时，统一使用 Mermaid 表达流程图、类图和调用关系图。

目录结构如下：

- `02_方案设计与功能说明`：说明系统总体方案和教务核心功能模块。
- `03_程序框图`：说明系统包结构、实体类图和核心业务流程图。
- `04_主要程序清单与数据结构设计`：说明主要接口、数据成员、函数调用关系、数据库逻辑结构和物理结构。
- `05_程序运行测试与分析`：说明测试环境、测试数据、测试用例、边界场景、接口联调与运行结果。
- `06_结论与心得`：总结系统优点、不足和课程设计收获。

维护本文档时应遵守三个原则。第一，报告内容应与项目实际保持一致，不能把尚未独立实现的类或组件描述为已完成代码；例如权限拦截器、成绩控制器等可作为设计职责说明，但需要同时说明当前项目由已有控制器、服务类、前端路由守卫和异常处理类共同承担相应功能。第二，章节正文应以段落论述为主，表格只用于字段结构、接口说明和测试用例等适合比较的信息。第三，新增或调整章节后，应同步检查章节编号、标题名称、图表含义和数据库字段描述，避免报告与工程目录、数据库脚本和测试文件发生偏差。
