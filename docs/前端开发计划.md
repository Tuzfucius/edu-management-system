# 前端开发计划

本文档规划教务管理系统前端开发。当前前端采用 Vue 3 + Vite + Element Plus + Vue Router，目标是先完成可演示、可联调、结构清晰的后台管理界面，再逐步接入后端 `/api/...` 接口。

## 一、当前文件架构分析

项目根目录当前是前后端分离结构：

```text
edu-management-system/
├─ src/                 Spring Boot 后端源码
├─ sql/                 MySQL 建库、建表、模拟数据脚本
├─ docs/                项目计划、数据库设计、前端计划等文档
└─ frontend/            Vue 前端工程
```

前端目录当前结构：

```text
frontend/
├─ package.json         前端依赖和脚本
├─ vite.config.js       Vite 配置
├─ public/              静态公共资源
└─ src/
   ├─ main.js           Vue 入口，挂载 Element Plus 和路由
   ├─ App.vue           根组件，仅承载 router-view
   ├─ router/index.js   当前路由配置
   ├─ layout/
   │  └─ AdminLayout.vue 后台通用布局，包含侧边栏、顶部栏、主内容区
   └─ views/
      ├─ Login.vue      登录页，当前为前端模拟登录
      └─ Dashboard.vue  数据看板，当前使用静态模拟数据
```

当前已有基础界面骨架，但还处于早期阶段：

```text
1. 已有登录页、后台布局、数据看板三个基础页面。
2. 侧边栏已预留学生、教师、课程、统计报表入口，但对应路由和页面尚未创建。
3. 登录逻辑目前只做前端必填校验和跳转，尚未接入 AuthController。
4. Dashboard 当前是静态数字，后续应改为后端统计接口 + ECharts 图表。
5. src/style.css 仍保留部分 Vite 模板样式，后续应整理为系统级全局样式或删除不用内容。
```

## 二、前端开发原则

```text
1. 使用 Vue 3 组合式 API，页面逻辑尽量保持清晰。
2. 使用 Element Plus 作为主组件库，不重复造基础表格、表单、弹窗组件。
3. 后台系统风格保持简洁、稳定、便于扫描，不做营销式页面。
4. 页面优先服务课程演示：登录、角色权限、CRUD、选课成绩流程、统计报表必须清楚可见。
5. 前端先用 mock 数据搭建页面，再逐步替换为 axios API 请求。
6. 表单校验前后端都要做，前端负责即时提示，后端负责最终业务规则。
7. 路由、API、组件按模块拆分，避免把所有逻辑堆在 views 文件中。
```

建议逐步形成以下结构：

```text
frontend/src/
├─ api/                 axios 实例和各业务模块 API
├─ assets/              图片、图标等资源
├─ components/          可复用业务组件
├─ layout/              后台布局
├─ router/              路由表和权限守卫
├─ stores/              登录用户、角色、菜单等状态；可先不用 Pinia，简单场景用 localStorage
├─ utils/               日期、字典、格式化、请求错误处理
└─ views/               页面级组件
   ├─ auth/
   ├─ admin/
   ├─ teacher/
   └─ student/
```

## 四、公共页面与通用能力

### 1. 登录页 `/login`

功能：

```text
1. 用户名、密码、角色输入与校验。
2. 调用登录接口，保存用户信息、角色和登录状态。
3. 根据角色跳转到对应首页。
4. 展示登录失败、账号禁用、角色不匹配等错误。
```

接口依赖：

```text
POST /api/auth/login
POST /api/auth/logout
GET  /api/auth/current
```

### 2. 后台布局 `AdminLayout`

功能：

```text
1. 左侧菜单按角色动态显示。
2. 顶部显示当前用户、角色、退出登录按钮。
3. 主内容区承载各业务页面。
4. 后续可增加面包屑、折叠菜单、刷新按钮。
```

### 3. 通用 CRUD 页面能力

基础数据页面应保持一致交互：

```text
1. 顶部筛选区：关键字、状态、所属上级实体等。
2. 操作区：新增、刷新、批量操作可选。
3. 数据表格：分页、状态标签、编辑、停用。
4. 表单弹窗：新增和编辑共用表单。
5. 删除策略：前端按钮文案使用“停用”，与后端逻辑删除一致。
```

## 八、API 模块规划

建议在 `frontend/src/api/` 下按业务拆分：

```text
request.js              axios 实例、baseURL、错误处理
auth.js                 登录、退出、当前用户
college.js              学院接口
major.js                专业接口
department.js           教研室接口
classInfo.js            班级接口
user.js                 用户接口
student.js              学生接口
teacher.js              教师接口
course.js               课程接口
teachingTask.js         任课安排接口
studentCourse.js        选课成绩接口
teacherStudent.js       指导关系接口
report.js               统计报表接口
operationLog.js         操作日志接口
```

请求约定：

```text
1. 前端统一通过 axios 调用 `/api/...`。
2. 后端统一返回 Result 结构后，前端在 request.js 中集中处理 code、message、data。
3. 分页参数建议统一为 page、size、keyword、status。
4. 日期和时间在前端只做展示格式化，不改变后端原始值。
```

## 九、开发阶段计划

### 阶段 1：整理前端骨架

```text
1. 清理 Vite 模板残留样式。
2. 补全路由分组和 404 页面。
3. 拆分 admin、teacher、student 三类菜单。
4. 增加 axios request.js 和 mock 数据目录。
5. 建立通用表格、状态标签、表单弹窗的开发约定。
```

### 阶段 2：管理员基础数据页面

```text
1. 学院、专业、教研室、班级、课程 CRUD 页面。
2. 先使用本地 mock 数据完成界面和交互。
3. 后端 Controller 完成后逐个切换到真实接口。
```

### 阶段 3：账号、学生、教师页面

```text
1. 登录接入 Auth API。
2. 用户管理、学生管理、教师管理页面。
3. 新增学生和教师时支持同步创建账号。
4. 增加角色路由守卫。
```

### 阶段 4：任课、选课、成绩流程

```text
1. 管理员维护任课安排。
2. 学生可选课程、我的课程、退课。
3. 教师查看课程学生并录入成绩。
4. 管理员查看和修正选课成绩。
```

### 阶段 5：统计报表和答辩优化

```text
1. Dashboard 接入真实统计接口。
2. 使用 ECharts 完成关键报表。
3. 完善加载态、空状态、错误提示。
4. 准备课程报告截图所需页面。
```

## 十、近期优先级

当前后端还没有 Controller 和 `/api/...` 接口，因此前端近期建议先做不依赖接口的结构建设：

```text
1. 先补齐路由、菜单、页面占位和统一布局。
2. 优先完成管理员端学院、专业、班级、学生、教师、课程页面的静态表格和表单。
3. 等后端 Result、AuthController 和基础 Controller 完成后，集中接入 axios。
4. 不要先做复杂权限框架；先用 localStorage 保存当前用户和角色，后续再替换为后端会话。
```

前后端最小闭环建议：

```text
登录页提交账号
    ↓
后端返回用户角色
    ↓
前端进入对应角色首页
    ↓
管理员打开学生管理
    ↓
学生列表从 MySQL 查询并展示
    ↓
完成新增、编辑、停用学生
```

这个闭环完成后，再复制到教师、课程、任课、选课、成绩和报表模块。
# 前端开发计划更新记录

当前前端已经完成三类角色主流程的真实接口接入：

- 管理员端：组织管理、人员管理、课程库、任课安排、统计报表。
- 教师端：我的任课、成绩录入。
- 学生端：可选课程、退课、我的成绩。

后续前端重点不再是静态页面补齐，而是课表视图、分页筛选、批量操作、导入导出、异常页和空状态优化。
