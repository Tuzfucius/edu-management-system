# 文档目录说明

本目录保存教务管理系统的需求、课程任务和设计规划文档，是后续编码和答辩材料的主要依据。

## 文件说明

- `java任务书.txt`：课程设计题目、业务描述和评分标准。
- `整体计划.md`：系统技术路线、模块拆分、开发顺序和页面规划。
- `数据库计划.md`：数据库实体、关系、字段和设计原则。
- `当前计划.md`：当前后端真实进度、已完成模块和后续后端实现顺序。
- `前端开发计划.md`：Vue + Element Plus 前端架构、页面清单、角色功能和分阶段开发计划。

## 使用规则

- 如果 `整体计划.md` 与 `数据库计划.md` 存在出入，以更具体的 `数据库计划.md` 为准。
- 如果旧文档仍提到 Thymeleaf、Bootstrap 或“暂时不要做前端页面”，以前端开发阶段的实际选择为准：当前前端采用 Vue 3 + Vite + Element Plus + Vue Router。
- 如果实现方案与课程评分点冲突，以 `java任务书.txt` 的要求为准。
- 新增设计文档时，文件名应能直接表达主题，例如 `权限设计.md`、`接口设计.md`、`报表设计.md`。
- 文档面向课程开发和智能体协作，优先使用简体中文，代码、表名、字段名和专业术语可保留英文。
